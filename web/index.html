ProFlame 2 ESPHome

Flash your LilyGo T-Embed CC1101 directly from this page. No command line, no Docker, no ESPHome install.

⭐ LilyGo T-Embed CC1101

Recommended hardware: ESP32-S3 + CC1101 radio + 1.9″ LCD + rotary encoder + 8-pixel WS2812 strip + Li-ion battery on a single board. Buy from LilyGo →

  1. Plug the T-Embed into this computer with a USB-C cable.
  2. Click Install below.
  3. Pick the serial port labeled USB JTAG/serial debug unit.
  4. Wait ~60 s — the page does the rest, then prompts for your Wi-Fi.
  5. Long-press the encoder, point your OEM remote at the device, and confirm to pair.
Web Serial API isn't available in this browser. Use Chrome or Edge on a desktop computer. Web Serial blocked — check your browser's site settings or try a different browser.

After flash completes, the page will show a Connect to Wi-Fi prompt. Enter your network SSID and password — the firmware uses Improv-Wi-Fi over the same USB connection to receive them. No SSID is hard-coded into the firmware.

What you get after flashing

  • Home Assistant entity set — climate (HEAT/OFF thermostat with fan modes), light (1–6 brightness as 0–100 %), all the proflame2 controls, plus a diagnostic block showing your captured serial & ECC.
  • Standalone LCD UI — drives the fireplace from the device with no Wi-Fi or HA dependency. Idle backlight dim, settings cog, info screen.
  • Fire-effect LED strip — bottom-edge WS2812 with red/orange/yellow flicker that runs while the burner is on. Killable from HA or the device.
  • Battery monitoring via the on-board BQ27220 fuel gauge.
  • On-device pairing — capture your OEM remote's serial & ECC directly through the encoder UI; no SDR needed.

Already running another build?

Web-flash will fully erase the existing firmware. If you've customized a YAML on top of this project, OTA-reflash from your existing setup instead — see the installation docs.
